# Laboratorio 20: Flujo CI/CD Completo — De Código Java a Producción

> **Objetivo:** Construir un pipeline CI/CD completo e integrado que cubra todas las fases: compilar una aplicación Java, ejecutar tests, construir imagen Docker, publicar en un registro, desplegar en Kubernetes y ejecutar pruebas de humo (smoke tests). Todo orquestado localmente con scripts y Docker.

> **Requisitos previos:**
> - Docker Desktop con Kubernetes habilitado
> - Terminal PowerShell
> - kubectl configurado

## 2️⃣ Código fuente de la aplicación

Crea `src/App.java`:

```java
import com.sun.net.httpserver.HttpServer;
import java.net.InetSocketAddress;
import java.time.LocalDateTime;
import java.util.Map;

public class App {
    static final String VERSION = "1.0.0";

    public static void main(String[] args) throws Exception {
        int port = Integer.parseInt(System.getenv().getOrDefault("APP_PORT", "8080"));
        HttpServer server = HttpServer.create(new InetSocketAddress(port), 0);

        server.createContext("/", ex -> {
            String host = java.net.InetAddress.getLocalHost().getHostName();
            String body = String.format(
                "{\"app\":\"cicd-demo\",\"version\":\"%s\",\"host\":\"%s\",\"time\":\"%s\"}",
                VERSION, host, LocalDateTime.now()
            );
            byte[] r = body.getBytes();
            ex.getResponseHeaders().set("Content-Type", "application/json");
            ex.sendResponseHeaders(200, r.length);
            ex.getResponseBody().write(r);
            ex.getResponseBody().close();
        });

        server.createContext("/health", ex -> {
            byte[] r = "{\"status\":\"UP\"}".getBytes();
            ex.getResponseHeaders().set("Content-Type", "application/json");
            ex.sendResponseHeaders(200, r.length);
            ex.getResponseBody().write(r);
            ex.getResponseBody().close();
        });

        server.start();
        System.out.printf("App v%s escuchando en puerto %d%n", VERSION, port);
    }
}
```

## 3️⃣ Tests unitarios (sin frameworks externos)

Crea `test/AppTest.java`:

```java
public class AppTest {
    static int passed = 0;
    static int failed = 0;

    public static void main(String[] args) {
        testVersionNotNull();
        testVersionFormat();
        testPortDefault();

        System.out.println("---------------------------");
        System.out.printf("Resultado: %d passed, %d failed%n", passed, failed);

        if (failed > 0) {
            System.exit(1);
        }
    }

    static void testVersionNotNull() {
        try {
            assert App.VERSION != null : "VERSION no debe ser null";
            assert !App.VERSION.isEmpty() : "VERSION no debe estar vacía";
            System.out.println("[PASS] testVersionNotNull");
            passed++;
        } catch (AssertionError e) {
            System.out.println("[FAIL] testVersionNotNull: " + e.getMessage());
            failed++;
        }
    }

    static void testVersionFormat() {
        try {
            String v = App.VERSION;
            assert v.matches("\\d+\\.\\d+\\.\\d+") : "VERSION debe ser semver (x.y.z), es: " + v;
            System.out.println("[PASS] testVersionFormat");
            passed++;
        } catch (AssertionError e) {
            System.out.println("[FAIL] testVersionFormat: " + e.getMessage());
            failed++;
        }
    }

    static void testPortDefault() {
        try {
            String port = System.getenv().getOrDefault("APP_PORT", "8080");
            int p = Integer.parseInt(port);
            assert p > 0 && p < 65536 : "Puerto fuera de rango: " + p;
            System.out.println("[PASS] testPortDefault");
            passed++;
        } catch (Exception e) {
            System.out.println("[FAIL] testPortDefault: " + e.getMessage());
            failed++;
        }
    }
}
```

## 4️⃣ Dockerfile multi-stage

Crea `Dockerfile`:

```dockerfile
# --- Etapa 1: Compilar ---
FROM eclipse-temurin:17-jdk-alpine AS build
WORKDIR /build
COPY src/App.java .
RUN javac App.java

# --- Etapa 2: Imagen final ---
FROM eclipse-temurin:17-jre-alpine
WORKDIR /app
COPY --from=build /build/App.class .
ENV APP_PORT=8080
EXPOSE 8080
HEALTHCHECK --interval=10s --timeout=3s --retries=3 \
    CMD wget -qO- http://localhost:8080/health || exit 1
CMD ["java", "App"]
```

  name: cicd-demo

      maxUnavailable: 0

      labels:
        app: cicd-demo

          image: cicd-demo:latest
          imagePullPolicy: Never

            - containerPort: 8080
          env:
            - name: APP_PORT

            httpGet:
              path: /health

            periodSeconds: 5

            httpGet:
              path: /health
              port: 8080

## 6️⃣ Script del pipeline CI/CD completo

Crea `pipeline.ps1`:

```powershell
# =============================================
#  PIPELINE CI/CD COMPLETO - Aplicación Java
# =============================================

$ErrorActionPreference = "Stop"
$IMAGE_NAME = "cicd-demo"
$IMAGE_TAG = "1.0.0"
$NAMESPACE = "cicd-demo"

function Write-Stage($name) {
    Write-Host ""
    Write-Host "=============================================" -ForegroundColor Cyan
    Write-Host "  STAGE: $name" -ForegroundColor Cyan
    Write-Host "=============================================" -ForegroundColor Cyan
}

# ===========================================
#  STAGE 1: COMPILACIÓN
# ===========================================
Write-Stage "1/7 - COMPILACIÓN"

Write-Host "Compilando App.java..."
if (!(Test-Path "out")) { New-Item -ItemType Directory -Path "out" | Out-Null }
docker run --rm -v "${PWD}/src:/src" -v "${PWD}/out:/out" eclipse-temurin:17-jdk-alpine sh -c "cd /src && javac -d /out App.java"

if ($LASTEXITCODE -ne 0) { Write-Host "ERROR: Compilación fallida" -ForegroundColor Red; exit 1 }
Write-Host "Compilación exitosa" -ForegroundColor Green

# ===========================================
#  STAGE 2: TESTS
# ===========================================
Write-Stage "2/7 - TESTS UNITARIOS"

Write-Host "Compilando y ejecutando tests..."
docker run --rm -v "${PWD}/src:/src" -v "${PWD}/test:/test" eclipse-temurin:17-jdk-alpine sh -c "
    cp /src/App.java /tmp/ && cp /test/AppTest.java /tmp/ &&
    cd /tmp && javac -ea App.java AppTest.java &&
    java -ea AppTest
"

if ($LASTEXITCODE -ne 0) { Write-Host "ERROR: Tests fallidos" -ForegroundColor Red; exit 1 }
Write-Host "Todos los tests pasaron" -ForegroundColor Green

# ===========================================
#  STAGE 3: ANÁLISIS DE CÓDIGO (LINT)
# ===========================================
Write-Stage "3/7 - ANÁLISIS ESTÁTICO"

Write-Host "Verificando buenas prácticas..."
$sourceContent = Get-Content "src/App.java" -Raw

$checks = @(
    @{ Name="Sin System.exit() en código principal"; Pattern="System\.exit\("; ShouldFind=$false },
    @{ Name="Tiene health check endpoint"; Pattern="/health"; ShouldFind=$true },
    @{ Name="Usa variable de entorno para puerto"; Pattern="APP_PORT"; ShouldFind=$true }
)

$lintPassed = $true
foreach ($check in $checks) {
    $found = $sourceContent -match $check.Pattern
    if ($found -eq $check.ShouldFind) {
        Write-Host "  [PASS] $($check.Name)" -ForegroundColor Green
    } else {
        Write-Host "  [FAIL] $($check.Name)" -ForegroundColor Red
        $lintPassed = $false
    }
}

if (!$lintPassed) { Write-Host "ERROR: Análisis estático fallido" -ForegroundColor Red; exit 1 }
Write-Host "Análisis estático OK" -ForegroundColor Green

# ===========================================
#  STAGE 4: BUILD DOCKER IMAGE
# ===========================================
Write-Stage "4/7 - BUILD IMAGEN DOCKER"

Write-Host "Construyendo imagen ${IMAGE_NAME}:${IMAGE_TAG}..."
docker build -t "${IMAGE_NAME}:${IMAGE_TAG}" -t "${IMAGE_NAME}:latest" .

if ($LASTEXITCODE -ne 0) { Write-Host "ERROR: Build fallido" -ForegroundColor Red; exit 1 }
Write-Host "Imagen construida: ${IMAGE_NAME}:${IMAGE_TAG}" -ForegroundColor Green

# ===========================================
#  STAGE 5: TEST DE LA IMAGEN
# ===========================================
Write-Stage "5/7 - TEST DE IMAGEN DOCKER"

Write-Host "Probando la imagen en contenedor..."
docker run -d --name cicd-test -p 9999:8080 "${IMAGE_NAME}:${IMAGE_TAG}"

Write-Host "Esperando que la aplicación arranque..."
Start-Sleep -Seconds 5

try {
    $response = Invoke-RestMethod -Uri "http://localhost:9999/health" -Method GET
    if ($response.status -eq "UP") {
        Write-Host "Health check OK: $($response | ConvertTo-Json -Compress)" -ForegroundColor Green
    } else {
        throw "Health check falló"
    }

    $appResponse = Invoke-RestMethod -Uri "http://localhost:9999/" -Method GET
    Write-Host "App response: $($appResponse | ConvertTo-Json -Compress)" -ForegroundColor Green
} catch {
    Write-Host "ERROR: Test de imagen fallido - $_" -ForegroundColor Red
    docker rm -f cicd-test 2>$null
    exit 1
} finally {
    docker rm -f cicd-test 2>$null
}

# ===========================================
#  STAGE 6: DEPLOY A KUBERNETES
# ===========================================
Write-Stage "6/7 - DEPLOY A KUBERNETES"

Write-Host "Aplicando manifiestos..."
kubectl apply -f k8s/namespace.yaml
kubectl apply -f k8s/deployment.yaml
kubectl apply -f k8s/service.yaml

Write-Host "Esperando a que los pods estén listos..."
kubectl rollout status deployment/cicd-demo -n $NAMESPACE --timeout=120s

if ($LASTEXITCODE -ne 0) { Write-Host "ERROR: Deploy fallido" -ForegroundColor Red; exit 1 }

kubectl get pods -n $NAMESPACE
Write-Host "Deploy completado" -ForegroundColor Green

# ===========================================
#  STAGE 7: SMOKE TESTS
# ===========================================
Write-Stage "7/7 - SMOKE TESTS EN KUBERNETES"

Write-Host "Esperando que el servicio esté accesible..."
Start-Sleep -Seconds 10

$smokeOK = $true
for ($i = 1; $i -le 5; $i++) {
    try {
        $result = Invoke-RestMethod -Uri "http://localhost:30090/health" -Method GET -TimeoutSec 5
        if ($result.status -eq "UP") {
            Write-Host "  Smoke test $i/5: OK (host=$($result.host // 'N/A'))" -ForegroundColor Green
        } else {
            Write-Host "  Smoke test $i/5: FAIL" -ForegroundColor Red
            $smokeOK = $false
        }
    } catch {
        Write-Host "  Smoke test $i/5: ERROR - $_" -ForegroundColor Red
        $smokeOK = $false
    }
    Start-Sleep -Seconds 1
}

if (!$smokeOK) {
    Write-Host ""
    Write-Host "SMOKE TESTS FALLIDOS - Iniciando rollback..." -ForegroundColor Red
    kubectl rollout undo deployment/cicd-demo -n $NAMESPACE
    exit 1
}

# ===========================================
#  RESULTADO FINAL
# ===========================================
Write-Host ""
Write-Host "=============================================" -ForegroundColor Green
Write-Host "  PIPELINE COMPLETADO CON ÉXITO" -ForegroundColor Green
Write-Host "=============================================" -ForegroundColor Green
Write-Host ""
Write-Host "  App: ${IMAGE_NAME}:${IMAGE_TAG}"
Write-Host "  URL: http://localhost:30090"
Write-Host "  Namespace: $NAMESPACE"
Write-Host "  Pods:" 
kubectl get pods -n $NAMESPACE -o custom-columns="NAME:.metadata.name,STATUS:.status.phase,READY:.status.conditions[?(@.type=='Ready')].status" --no-headers
```

## ✅ Resumen del Pipeline

| Etapa | Descripción | Herramienta |
|-------|-------------|-------------|
| **1. Compilación** | Compila el código Java | Docker + JDK 17 |
| **2. Tests** | Ejecuta tests unitarios | Docker + JDK 17 |
| **3. Análisis estático** | Verifica buenas prácticas | PowerShell |
| **4. Build imagen** | Construye imagen Docker multi-stage | Docker |
| **5. Test imagen** | Prueba la imagen en contenedor | Docker + curl |
| **6. Deploy K8s** | Despliega en Kubernetes con rolling update | kubectl |
| **7. Smoke tests** | Verifica la app en producción | PowerShell + HTTP |

Este pipeline cubre el flujo CI/CD completo visto a lo largo del curso: desde el código fuente hasta la aplicación corriendo en Kubernetes, con validaciones en cada paso y rollback automático si los smoke tests fallan.
