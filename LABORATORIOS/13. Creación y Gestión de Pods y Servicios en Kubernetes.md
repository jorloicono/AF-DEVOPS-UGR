# Laboratorio 13: Creación y Gestión de Pods y Servicios en Kubernetes

> **Objetivo:** Aprender a crear, inspeccionar y gestionar Pods y Services en Kubernetes, exponiendo una aplicación Java al exterior del clúster.

> **Requisitos previos:**
> - Docker Desktop con Kubernetes habilitado (ver Laboratorio 12)
> - `kubectl` instalado y configurado

---

## 1️⃣ Crear la aplicación Java

Crea una carpeta para el proyecto:

```powershell
mkdir k8s-pods-lab
cd k8s-pods-lab
```

Crea el archivo `App.java`:

```java
import com.sun.net.httpserver.HttpServer;
import java.net.InetSocketAddress;

public class App {
    public static void main(String[] args) throws Exception {
        String hostname = java.net.InetAddress.getLocalHost().getHostName();
        int port = 8080;

        HttpServer server = HttpServer.create(new InetSocketAddress(port), 0);

        server.createContext("/", exchange -> {
            String body = "Hola desde el Pod: " + hostname;
            byte[] r = body.getBytes();
            exchange.sendResponseHeaders(200, r.length);
            exchange.getResponseBody().write(r);
            exchange.getResponseBody().close();
        });

        server.createContext("/health", exchange -> {
            byte[] r = "OK".getBytes();
            exchange.sendResponseHeaders(200, r.length);
            exchange.getResponseBody().write(r);
            exchange.getResponseBody().close();
        });

        server.start();
        System.out.println("Servidor escuchando en puerto " + port + " | Pod: " + hostname);
    }
}
```

## 2️⃣ Construir la imagen Docker

Crea el archivo `Dockerfile`:

```dockerfile
FROM eclipse-temurin:17-jdk-alpine AS build
WORKDIR /app
COPY App.java .
RUN javac App.java

FROM eclipse-temurin:17-jre-alpine
WORKDIR /app
COPY --from=build /app/App.class .
EXPOSE 8080
CMD ["java", "App"]
```

```powershell
docker build -t java-pods-app:1.0 .
```

---

## 3️⃣ Crear un Pod básico con YAML

Crea el archivo `pod-basico.yaml`:

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: java-app-v1
  labels:
    app: java-app
    version: v1
spec:
  containers:
    - name: java-app
      image: java-pods-app:1.0
      imagePullPolicy: Never
      ports:
        - containerPort: 8080
```

> **Nota:** `imagePullPolicy: Never` indica a Kubernetes que use la imagen local de Docker.

Aplica el manifiesto:

```powershell
kubectl apply -f pod-basico.yaml
```

## 4️⃣ Inspeccionar el Pod

Verifica que el Pod está corriendo:

```powershell
kubectl get pods
```

Consulta información detallada del Pod:

```powershell
kubectl describe pod java-app-v1
```

**Campos importantes del `describe`:**

| Campo | Descripción |
|-------|-------------|
| `Status` | Estado actual: Pending, Running, Succeeded, Failed |
| `IP` | Dirección IP interna del Pod en el clúster |
| `Containers` | Lista de contenedores y su estado |
| `Events` | Historial de eventos (pull, create, start) |

Ver los logs del contenedor:

```powershell
kubectl logs java-app-v1
```

Ejecutar un comando dentro del Pod:

```powershell
kubectl exec java-app-v1 -- hostname
kubectl exec -it java-app-v1 -- sh
```

> Dentro del shell puedes ejecutar `wget -qO- http://localhost:8080` para probar la app. Sal con `exit`.

## 5️⃣ Acceder al Pod con port-forward

El Pod solo es accesible desde dentro del clúster. Usa port-forward para probarlo desde tu máquina:

```powershell
kubectl port-forward java-app-v1 8080:8080
```

Abre `http://localhost:8080` en el navegador. Verás el nombre del Pod. Detén con `Ctrl+C`.

## 6️⃣ Crear un Pod multi-contenedor

Crea el archivo `pod-multi.yaml`:

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: java-app-multi
  labels:
    app: java-app-multi
spec:
  containers:
    - name: java-app
      image: java-pods-app:1.0
      imagePullPolicy: Never
      ports:
        - containerPort: 8080

    - name: sidecar-logger
      image: busybox
      command: ["sh", "-c"]
      args:
        - |
          while true; do
            echo "[$(date)] Verificando app..."
            wget -qO- http://localhost:8080/health && echo " -> App OK" || echo " -> App CAIDA"
            sleep 10
          done
```

```powershell
kubectl apply -f pod-multi.yaml
```

**Explicación:** Ambos contenedores comparten la misma red (localhost). El sidecar monitorea la app principal cada 10 segundos.

Ver los logs de cada contenedor:

```powershell
kubectl logs java-app-multi -c java-app
kubectl logs java-app-multi -c sidecar-logger
```

## 7️⃣ Crear un Service ClusterIP

Un **Service** proporciona una dirección estable para acceder a los Pods. Crea `service-clusterip.yaml`:

```yaml
apiVersion: v1
kind: Service
metadata:
  name: java-app-clusterip
spec:
  type: ClusterIP
  selector:
    app: java-app
  ports:
    - port: 80
      targetPort: 8080
```

```powershell
kubectl apply -f service-clusterip.yaml
kubectl get services
```

**Explicación:** `ClusterIP` es el tipo por defecto. Solo accesible dentro del clúster. El `selector` dirige el tráfico a Pods con label `app: java-app`.

Prueba desde dentro del clúster:

```powershell
kubectl run test-curl --rm -it --image=busybox --restart=Never -- wget -qO- http://java-app-clusterip
```

## 8️⃣ Crear un Service NodePort

Para acceder desde fuera del clúster, usa NodePort. Crea `service-nodeport.yaml`:

```yaml
apiVersion: v1
kind: Service
metadata:
  name: java-app-nodeport
spec:
  type: NodePort
  selector:
    app: java-app
  ports:
    - port: 80
      targetPort: 8080
      nodePort: 30080
```

```powershell
kubectl apply -f service-nodeport.yaml
kubectl get services
```

Abre `http://localhost:30080` en el navegador. El Service NodePort expone la app en el puerto 30080 de tu máquina.

**Comparativa de tipos de Service:**

| Tipo | Acceso | Uso típico |
|------|--------|------------|
| `ClusterIP` | Solo dentro del clúster | Comunicación entre servicios internos |
| `NodePort` | Desde fuera del clúster (puerto fijo) | Desarrollo y pruebas |
| `LoadBalancer` | IP externa (nube) | Producción en cloud |

## 9️⃣ Gestión del ciclo de vida de Pods

Reiniciar un Pod (eliminarlo para que un Deployment lo recree):

```powershell
kubectl delete pod java-app-v1
```

> **Nota:** Un Pod eliminado directamente no se recrea. Para auto-recuperación necesitas un Deployment (ver Lab 15).

Ver Pods en todos los namespaces:

```powershell
kubectl get pods --all-namespaces
```

Filtrar Pods por etiqueta:

```powershell
kubectl get pods -l app=java-app
kubectl get pods -l version=v1
```
