> **Objetivo:** Aprender a escribir Playbooks de Ansible para automatizar tareas complejas: compilar y desplegar una aplicación Java, usar handlers, templates Jinja2, condicionales, bucles y organizar tareas con roles.

## 1️⃣ Preparar el entorno

Crea la carpeta del proyecto:

```powershell
mkdir ansible-playbooks-lab
cd ansible-playbooks-lab
```

Crea `Dockerfile.node`:

```dockerfile
FROM eclipse-temurin:17-jdk-alpine

RUN apk add --no-cache openssh python3 bash curl && \
    ssh-keygen -A && \
    adduser -D devops && \
    echo "devops:devops123" | chpasswd && \
    echo "devops ALL=(ALL) NOPASSWD:ALL" >> /etc/sudoers && \
    sed -i 's/#PermitRootLogin.*/PermitRootLogin no/' /etc/ssh/sshd_config && \
    sed -i 's/#PasswordAuthentication.*/PasswordAuthentication yes/' /etc/ssh/sshd_config

EXPOSE 22
CMD ["/usr/sbin/sshd", "-D"]
```

Crea `Dockerfile.control`:

```dockerfile
FROM python:3.11-alpine

RUN apk add --no-cache openssh-client sshpass bash && \
    pip install --no-cache-dir ansible

RUN mkdir -p /root/.ssh && \
    ssh-keygen -t rsa -b 2048 -f /root/.ssh/id_rsa -N "" && \
    echo "StrictHostKeyChecking no" > /root/.ssh/config

WORKDIR /ansible
CMD ["sleep", "infinity"]
```

Crea `docker-compose.yml`:

```yaml
services:
  control:
    build:
      context: .
      dockerfile: Dockerfile.control
    container_name: ansible-control
    hostname: control
    volumes:
      - ./ansible:/ansible
    networks:
      - ansible-net
    depends_on:
      - web1
      - web2

  web1:
    build:
      context: .
      dockerfile: Dockerfile.node
    container_name: ansible-web1
    hostname: web1
    networks:
      - ansible-net

  web2:
    build:
      context: .
      dockerfile: Dockerfile.node
    container_name: ansible-web2
    hostname: web2
    networks:
      - ansible-net

networks:
  ansible-net:
    driver: bridge
```

```powershell
mkdir ansible
docker compose up -d --build
```

Crea `ansible/inventory.ini`:

```ini
[web]
web1 ansible_host=web1
web2 ansible_host=web2

[web:vars]
ansible_user=devops
ansible_password=devops123
ansible_connection=ssh
```

Verifica conectividad:

```powershell
docker exec ansible-control ansible -i /ansible/inventory.ini web -m ping
```

## 2️⃣ Tu primer Playbook

Crea `ansible/playbook-basico.yml`:

```yaml
---
- name: Primer Playbook - Información del sistema
  hosts: web
  gather_facts: true
  tasks:
    - name: Mostrar el hostname
      debug:
        msg: "Hostname: {{ ansible_hostname }}"

    - name: Mostrar la IP
      debug:
        msg: "IP: {{ ansible_default_ipv4.address }}"

    - name: Mostrar versión de Java
      command: java -version
      register: java_output
      changed_when: false

    - name: Imprimir versión de Java
      debug:
        msg: "{{ java_output.stderr_lines }}"
```

Ejecútalo:

```powershell
docker exec ansible-control ansible-playbook -i /ansible/inventory.ini /ansible/playbook-basico.yml
```

**Anatomía de un Playbook:**

| Elemento | Descripción |
|----------|-------------|
| `name` | Nombre descriptivo del play/tarea |
| `hosts` | Grupo del inventario donde ejecutar |
| `gather_facts` | Recopilar información del sistema |
| `tasks` | Lista de tareas a ejecutar en orden |
| `register` | Guardar la salida en una variable |
| `debug` | Mostrar mensajes/variables |

## 3️⃣ Playbook para compilar y desplegar una aplicación Java

Crea la aplicación Java. Crea `ansible/files/App.java`:

```powershell
mkdir ansible/files
```

```java
import com.sun.net.httpserver.HttpServer;
import java.net.InetSocketAddress;

public class App {
    public static void main(String[] args) throws Exception {
        String host = java.net.InetAddress.getLocalHost().getHostName();
        HttpServer server = HttpServer.create(new InetSocketAddress(8080), 0);

        server.createContext("/", exchange -> {
            String body = "Hola desde " + host + " - Desplegado con Ansible!";
            byte[] r = body.getBytes();
            exchange.sendResponseHeaders(200, r.length);
            exchange.getResponseBody().write(r);
            exchange.getResponseBody().close();
        });

        server.start();
        System.out.println("Servidor iniciado en " + host + ":8080");
    }
}
```

Crea `ansible/deploy-java.yml`:

```yaml
---
- name: Compilar y desplegar aplicación Java
  hosts: web
  become: true
  tasks:
    - name: Crear directorio de la aplicación
      file:
        path: /opt/java-app
        state: directory
        owner: devops
        mode: '0755'

    - name: Copiar código fuente
      copy:
        src: files/App.java
        dest: /opt/java-app/App.java
        owner: devops
      register: source_changed

    - name: Compilar la aplicación
      command: javac /opt/java-app/App.java
      args:
        chdir: /opt/java-app
      become_user: devops
      when: source_changed.changed

    - name: Verificar que el .class existe
      stat:
        path: /opt/java-app/App.class
      register: class_file

    - name: Confirmar compilación
      debug:
        msg: "App.class existe: {{ class_file.stat.exists }}, tamaño: {{ class_file.stat.size }} bytes"
```

```powershell
docker exec ansible-control ansible-playbook -i /ansible/inventory.ini /ansible/deploy-java.yml
```

Ejecuta una segunda vez y observa que Ansible es **idempotente**: las tareas que no cambian aparecen como `ok` en vez de `changed`.

## 4️⃣ Handlers (ejecutar tareas cuando algo cambia)

Crea `ansible/deploy-con-handler.yml`:

```yaml
---
- name: Desplegar con reinicio automático
  hosts: web
  become: true
  tasks:
    - name: Detener aplicación si está corriendo
      shell: "pkill -f 'java App' || true"
      changed_when: false

    - name: Copiar código fuente
      copy:
        src: files/App.java
        dest: /opt/java-app/App.java
        owner: devops
      notify: Recompilar y reiniciar

    - name: Crear script de arranque
      copy:
        content: |
          #!/bin/bash
          cd /opt/java-app
          nohup java App > /var/log/java-app.log 2>&1 &
          echo $! > /var/run/java-app.pid
        dest: /opt/java-app/start.sh
        mode: '0755'

  handlers:
    - name: Recompilar y reiniciar
      shell: |
        cd /opt/java-app
        javac App.java
        pkill -f 'java App' || true
        sleep 1
        nohup java App > /var/log/java-app.log 2>&1 &
```

```powershell
docker exec ansible-control ansible-playbook -i /ansible/inventory.ini /ansible/deploy-con-handler.yml
```

> **Handler:** Solo se ejecuta si la tarea que lo notifica reporta un cambio (`changed`). Si ejecutas el Playbook de nuevo sin cambiar App.java, el handler **no** se ejecuta.

Verifica que la app corre:

```powershell
docker exec ansible-web1 curl -s http://localhost:8080
docker exec ansible-web2 curl -s http://localhost:8080
```

## 5️⃣ Templates con Jinja2

Los templates permiten generar archivos dinámicos con variables.

Crea `ansible/templates/app-config.properties.j2`:

```powershell
mkdir ansible/templates
```

```
# Configuración generada por Ansible
app.name={{ app_name }}
app.version={{ app_version }}
app.host={{ ansible_hostname }}
app.ip={{ ansible_default_ipv4.address }}
app.env={{ app_env | default('desarrollo') }}
app.port={{ app_port }}
app.fecha_despliegue={{ ansible_date_time.iso8601 }}
```

Crea `ansible/deploy-template.yml`:

```yaml
---
- name: Desplegar configuración con Template
  hosts: web
  become: true
  vars:
    app_name: "MiAppJava"
    app_version: "1.0.0"
    app_port: 8080
  tasks:
    - name: Generar archivo de configuración
      template:
        src: templates/app-config.properties.j2
        dest: /opt/java-app/app-config.properties
        owner: devops

    - name: Mostrar configuración generada
      command: cat /opt/java-app/app-config.properties
      register: config_output
      changed_when: false

    - name: Ver configuración
      debug:
        msg: "{{ config_output.stdout_lines }}"
```

```powershell
docker exec ansible-control ansible-playbook -i /ansible/inventory.ini /ansible/deploy-template.yml
```

Observa cómo cada nodo tiene valores diferentes para `ansible_hostname` y `ansible_default_ipv4.address`.

## 6️⃣ Condicionales y bucles

Crea `ansible/condicionales-bucles.yml`:

```yaml
---
- name: Condicionales y Bucles
  hosts: web
  become: true
  vars:
    paquetes_necesarios:
      - curl
      - git
      - vim
    crear_usuarios:
      - { nombre: app1, shell: /bin/bash }
      - { nombre: app2, shell: /bin/bash }
  tasks:
    - name: Instalar paquetes necesarios
      command: "apk add --no-cache {{ item }}"
      loop: "{{ paquetes_necesarios }}"
      register: install_result
      changed_when: "'Installing' in install_result.stdout"

    - name: Crear directorios para cada usuario
      file:
        path: "/opt/apps/{{ item.nombre }}"
        state: directory
        mode: '0755'
      loop: "{{ crear_usuarios }}"

    - name: Verificar espacio en disco
      shell: "df -h / | tail -1 | awk '{print $5}' | sed 's/%//'"
      register: disk_usage
      changed_when: false

    - name: Advertir si disco > 80%
      debug:
        msg: "ADVERTENCIA: Disco al {{ disk_usage.stdout }}% en {{ ansible_hostname }}"
      when: disk_usage.stdout | int > 80

    - name: Todo bien con el disco
      debug:
        msg: "Disco en {{ ansible_hostname }} al {{ disk_usage.stdout }}% - OK"
      when: disk_usage.stdout | int <= 80
```

```powershell
docker exec ansible-control ansible-playbook -i /ansible/inventory.ini /ansible/condicionales-bucles.yml
```

## ✅ Resumen

En este laboratorio has aprendido a:

- Escribir **Playbooks** con tareas, variables y facts
- **Compilar y desplegar** una aplicación Java de forma automatizada
- Usar **handlers** para ejecutar acciones solo cuando algo cambia
- Crear archivos dinámicos con **templates Jinja2**
- Implementar **condicionales** (`when`) para lógica de decisión
- Usar **bucles** (`loop`) para repetir tareas con diferentes datos
