# Laboratorio 18: Gestión de Inventarios y Variables en Ansible

> **Objetivo:** Dominar la gestión avanzada de inventarios (estáticos, dinámicos, múltiples grupos), el sistema de precedencia de variables (group_vars, host_vars, extra vars) y la protección de datos sensibles con Ansible Vault.

> **Requisitos previos:**
> - Docker Desktop instalado
> - Terminal PowerShell

---

## 1️⃣ Preparar el entorno

Crea la carpeta del proyecto:

```powershell
mkdir ansible-inventarios-lab
cd ansible-inventarios-lab
```

Crea `Dockerfile.node`:

```dockerfile
FROM eclipse-temurin:17-jdk-alpine

RUN apk add --no-cache openssh python3 bash curl && \
    ssh-keygen -A && \
    adduser -D devops && \
    echo "devops:devops123" | chpasswd && \
    echo "devops ALL=(ALL) NOPASSWD:ALL" >> /etc/sudoers && \
    sed -i 's/#PasswordAuthentication.*/PasswordAuthentication yes/' /etc/ssh/sshd_config

EXPOSE 22
CMD ["/usr/sbin/sshd", "-D"]
```

Crea `Dockerfile.control`:

```dockerfile
FROM python:3.11-alpine

RUN apk add --no-cache openssh-client sshpass bash && \
    pip install --no-cache-dir ansible

RUN mkdir -p /root/.ssh && \
    ssh-keygen -t rsa -b 2048 -f /root/.ssh/id_rsa -N "" && \
    echo "StrictHostKeyChecking no" > /root/.ssh/config

WORKDIR /ansible
CMD ["sleep", "infinity"]
```

Crea `docker-compose.yml` con 4 nodos simulando diferentes entornos:

```yaml
services:
  control:
    build:
      context: .
      dockerfile: Dockerfile.control
    container_name: ansible-control
    hostname: control
    volumes:
      - ./ansible:/ansible
    networks:
      - ansible-net

  web-prod1:
    build: { context: ., dockerfile: Dockerfile.node }
    container_name: web-prod1
    hostname: web-prod1
    networks: [ansible-net]

  web-prod2:
    build: { context: ., dockerfile: Dockerfile.node }
    container_name: web-prod2
    hostname: web-prod2
    networks: [ansible-net]

  web-staging:
    build: { context: ., dockerfile: Dockerfile.node }
    container_name: web-staging
    hostname: web-staging
    networks: [ansible-net]

  db-prod:
    build: { context: ., dockerfile: Dockerfile.node }
    container_name: db-prod
    hostname: db-prod
    networks: [ansible-net]

networks:
  ansible-net:
    driver: bridge
```

```powershell
mkdir ansible
docker compose up -d --build
```

## 2️⃣ Inventario estático con múltiples grupos

Crea `ansible/inventory.ini`:

```ini
[web_produccion]
web-prod1 ansible_host=web-prod1
web-prod2 ansible_host=web-prod2

[web_staging]
web-staging ansible_host=web-staging

[base_datos]
db-prod ansible_host=db-prod

[web:children]
web_produccion
web_staging

[produccion:children]
web_produccion
base_datos

[todos:children]
web
base_datos

[todos:vars]
ansible_user=devops
ansible_password=devops123
ansible_connection=ssh
```

**Estructura de grupos:**

| Grupo | Nodos |
|-------|-------|
| `web_produccion` | web-prod1, web-prod2 |
| `web_staging` | web-staging |
| `base_datos` | db-prod |
| `web` (padre) | web_produccion + web_staging |
| `produccion` (padre) | web_produccion + base_datos |
| `todos` (padre) | web + base_datos |

Verifica el inventario:

```powershell
docker exec ansible-control ansible-inventory -i /ansible/inventory.ini --list
docker exec ansible-control ansible-inventory -i /ansible/inventory.ini --graph
```

Ejecuta ping a diferentes grupos:

```powershell
docker exec ansible-control ansible -i /ansible/inventory.ini web -m ping
docker exec ansible-control ansible -i /ansible/inventory.ini produccion -m ping
docker exec ansible-control ansible -i /ansible/inventory.ini web_staging -m ping
```

## 3️⃣ Inventario en formato YAML

Crea `ansible/inventory.yml`:

```yaml
all:
  children:
    web:
      children:
        web_produccion:
          hosts:
            web-prod1:
              ansible_host: web-prod1
              http_port: 8080
            web-prod2:
              ansible_host: web-prod2
              http_port: 8081
        web_staging:
          hosts:
            web-staging:
              ansible_host: web-staging
              http_port: 9090
    base_datos:
      hosts:
        db-prod:
          ansible_host: db-prod
          db_port: 5432
  vars:
    ansible_user: devops
    ansible_password: devops123
    ansible_connection: ssh
```

```powershell
docker exec ansible-control ansible-inventory -i /ansible/inventory.yml --graph
docker exec ansible-control ansible -i /ansible/inventory.yml all -m ping
```

## 4️⃣ Variables de grupo (group_vars)

Crea la estructura de directorios:

```powershell
mkdir ansible/group_vars
```

Crea `ansible/group_vars/all.yml` (variables para todos):

```yaml
---
app_name: "MiAppJava"
app_user: devops
java_home: /opt/java/openjdk
```

Crea `ansible/group_vars/web_produccion.yml`:

```yaml
---
app_env: produccion
app_port: 8080
log_level: WARN
max_memory: 512m
replicas: 2
```

Crea `ansible/group_vars/web_staging.yml`:

```yaml
---
app_env: staging
app_port: 9090
log_level: DEBUG
max_memory: 256m
replicas: 1
```

Crea `ansible/group_vars/base_datos.yml`:

```yaml
---
db_name: app_database
db_port: 5432
db_max_connections: 100
```

Crea un Playbook para verificar. Crea `ansible/mostrar-variables.yml`:

```yaml
---
- name: Mostrar variables por grupo
  hosts: all
  gather_facts: false
  tasks:
    - name: Mostrar configuración del grupo web
      debug:
        msg: >
          Host: {{ inventory_hostname }} |
          App: {{ app_name }} |
          Env: {{ app_env | default('N/A') }} |
          Puerto: {{ app_port | default(http_port | default('N/A')) }}
      when: "'web' in group_names"

    - name: Mostrar configuración de BD
      debug:
        msg: >
          Host: {{ inventory_hostname }} |
          BD: {{ db_name }} |
          Puerto: {{ db_port }}
      when: "'base_datos' in group_names"
```

```powershell
docker exec ansible-control ansible-playbook -i /ansible/inventory.ini /ansible/mostrar-variables.yml
```

Observa cómo cada nodo recibe las variables de su grupo específico.

## 5️⃣ Variables de host (host_vars)

Para configuración específica de un host:

```powershell
mkdir ansible/host_vars
```

Crea `ansible/host_vars/web-prod1.yml`:

```yaml
---
server_id: 1
is_primary: true
custom_jvm_args: "-Xms256m -Xmx512m"
```

Crea `ansible/host_vars/web-prod2.yml`:

```yaml
---
server_id: 2
is_primary: false
custom_jvm_args: "-Xms128m -Xmx256m"
```

Crea `ansible/verificar-host-vars.yml`:

```yaml
---
- name: Verificar host_vars
  hosts: web_produccion
  gather_facts: false
  tasks:
    - name: Mostrar config específica del host
      debug:
        msg: >
          {{ inventory_hostname }} -
          ID: {{ server_id }} |
          Primario: {{ is_primary }} |
          JVM: {{ custom_jvm_args }}
```

```powershell
docker exec ansible-control ansible-playbook -i /ansible/inventory.ini /ansible/verificar-host-vars.yml
```

**Precedencia de variables (de menor a mayor):**

| Prioridad | Fuente |
|-----------|--------|
| 1 (menor) | Valores por defecto del rol |
| 2 | `group_vars/all` |
| 3 | `group_vars/<grupo>` |
| 4 | `host_vars/<host>` |
| 5 | Vars en el Playbook (`vars:`) |
| 6 | Vars en tareas (`vars:`) |
| 7 | Extra vars (`-e`) — **siempre gana** |

## 6️⃣ Extra vars (máxima prioridad)

Las variables pasadas por línea de comandos tienen la prioridad más alta:

```powershell
docker exec ansible-control ansible-playbook -i /ansible/inventory.ini /ansible/mostrar-variables.yml -e "app_name=AppOverride app_env=testing"
```

Observa cómo `app_name` y `app_env` son sobreescritas en todos los hosts.

## 7️⃣ Ansible Vault (proteger datos sensibles)

### Crear un archivo cifrado

```powershell
docker exec -it ansible-control ansible-vault create /ansible/group_vars/secretos.yml
```

Cuando pida contraseña usa: `vault123`

En el editor que se abre, escribe:

```yaml
db_password: SuperSecreta2024
api_key: abcdef-123456-ghijkl
```

Guarda y sal (`:wq` en vi).

### Ver el archivo cifrado

```powershell
docker exec ansible-control cat /ansible/group_vars/secretos.yml
```

Verás texto cifrado. Para ver el contenido real:

```powershell
docker exec -it ansible-control ansible-vault view /ansible/group_vars/secretos.yml
```

### Usar variables cifradas en un Playbook

Crea `ansible/usar-vault.yml`:

```yaml
---
- name: Usar variables cifradas
  hosts: web_produccion
  gather_facts: false
  vars_files:
    - group_vars/secretos.yml
  tasks:
    - name: Mostrar que las variables están disponibles
      debug:
        msg: "DB Password tiene {{ db_password | length }} caracteres. API Key empieza con {{ api_key[:5] }}..."
```

Ejecútalo con la contraseña del vault:

```powershell
docker exec -it ansible-control ansible-playbook -i /ansible/inventory.ini /ansible/usar-vault.yml --ask-vault-pass
```

Ingresa `vault123` cuando lo pida.

### Cifrar un archivo existente

```powershell
docker exec -it ansible-control ansible-vault encrypt /ansible/host_vars/web-prod1.yml
```

Para editarlo cifrado:

```powershell
docker exec -it ansible-control ansible-vault edit /ansible/host_vars/web-prod1.yml
```

Para descifrarlo permanentemente:

```powershell
docker exec -it ansible-control ansible-vault decrypt /ansible/host_vars/web-prod1.yml
```

## 8️⃣ Playbook completo usando toda la jerarquía

Crea `ansible/despliegue-completo.yml`:

```yaml
---
- name: Despliegue usando jerarquía de variables
  hosts: web
  become: true
  tasks:
    - name: Crear directorio de la app
      file:
        path: "/opt/{{ app_name | lower | replace(' ', '-') }}"
        state: directory
        owner: "{{ app_user }}"
        mode: '0755'

    - name: Generar archivo de entorno
      copy:
        content: |
          APP_NAME={{ app_name }}
          APP_ENV={{ app_env | default('desarrollo') }}
          APP_PORT={{ app_port | default(http_port | default(8080)) }}
          LOG_LEVEL={{ log_level | default('INFO') }}
          MAX_MEMORY={{ max_memory | default('256m') }}
          JAVA_HOME={{ java_home }}
        dest: "/opt/{{ app_name | lower | replace(' ', '-') }}/.env"
        owner: "{{ app_user }}"

    - name: Mostrar configuración generada
      command: "cat /opt/{{ app_name | lower | replace(' ', '-') }}/.env"
      register: env_output
      changed_when: false

    - name: Ver configuración
      debug:
        msg: "{{ env_output.stdout_lines }}"
```

```powershell
docker exec ansible-control ansible-playbook -i /ansible/inventory.ini /ansible/despliegue-completo.yml
```

Observa cómo cada host recibe diferentes valores según su grupo y host_vars.

## ✅ Resumen

En este laboratorio has aprendido a:

- Crear **inventarios estáticos** con múltiples grupos y grupos padre (`children`)
- Usar inventarios en formato **INI** y **YAML**
- Definir variables a nivel de **grupo** (`group_vars/`) y de **host** (`host_vars/`)
- Entender la **precedencia de variables** de Ansible
- Usar **extra vars** (`-e`) para sobreescribir cualquier variable
- Proteger datos sensibles con **Ansible Vault** (crear, ver, editar, cifrar, descifrar)
- Combinar toda la jerarquía de variables en un despliegue realista
