# Laboratorio 14: ConfigMaps y Secrets en Kubernetes

> **Objetivo:** Aprender a externalizar la configuración de aplicaciones usando ConfigMaps y a gestionar datos sensibles con Secrets, inyectándolos en Pods como variables de entorno y como archivos montados.

> **Requisitos previos:**
> - Docker Desktop con Kubernetes habilitado
> - `kubectl` instalado
> - Imagen `java-pods-app:1.0` construida (ver Laboratorio 13), o construirla aquí

---

## 1️⃣ Crear la aplicación Java configurable

Crea la carpeta del proyecto:

```powershell
mkdir k8s-config-lab
cd k8s-config-lab
```

Crea el archivo `App.java`:

```java
import com.sun.net.httpserver.HttpServer;
import java.net.InetSocketAddress;
import java.nio.file.Files;
import java.nio.file.Path;

public class App {
    public static void main(String[] args) throws Exception {
        // Leer configuración desde variables de entorno
        String appName = System.getenv().getOrDefault("APP_NAME", "MiApp");
        String appEnv = System.getenv().getOrDefault("APP_ENV", "desarrollo");
        int port = Integer.parseInt(System.getenv().getOrDefault("APP_PORT", "8080"));

        // Leer secretos desde variables de entorno
        String dbUser = System.getenv().getOrDefault("DB_USER", "sin-usuario");
        String dbPass = System.getenv().getOrDefault("DB_PASSWORD", "sin-password");

        HttpServer server = HttpServer.create(new InetSocketAddress(port), 0);

        server.createContext("/", exchange -> {
            String body = String.format(
                "App: %s\nEntorno: %s\nPuerto: %d\nDB User: %s\nDB Pass: %s",
                appName, appEnv, port, dbUser, "***" + dbPass.substring(Math.max(0, dbPass.length() - 3))
            );
            byte[] r = body.getBytes();
            exchange.sendResponseHeaders(200, r.length);
            exchange.getResponseBody().write(r);
            exchange.getResponseBody().close();
        });

        // Endpoint que lee configuración desde archivo montado
        server.createContext("/file-config", exchange -> {
            String body;
            Path configPath = Path.of("/config/app.properties");
            if (Files.exists(configPath)) {
                body = "Configuración desde archivo:\n" + Files.readString(configPath);
            } else {
                body = "No se encontró /config/app.properties";
            }
            byte[] r = body.getBytes();
            exchange.sendResponseHeaders(200, r.length);
            exchange.getResponseBody().write(r);
            exchange.getResponseBody().close();
        });

        // Endpoint que lee secretos desde archivo montado
        server.createContext("/file-secret", exchange -> {
            String body;
            Path secretPath = Path.of("/secrets/credentials.txt");
            if (Files.exists(secretPath)) {
                body = "Secreto desde archivo:\n" + Files.readString(secretPath);
            } else {
                body = "No se encontró /secrets/credentials.txt";
            }
            byte[] r = body.getBytes();
            exchange.sendResponseHeaders(200, r.length);
            exchange.getResponseBody().write(r);
            exchange.getResponseBody().close();
        });

        server.createContext("/health", exchange -> {
            byte[] r = "OK".getBytes();
            exchange.sendResponseHeaders(200, r.length);
            exchange.getResponseBody().write(r);
            exchange.getResponseBody().close();
        });

        server.start();
        System.out.println("[" + appEnv + "] " + appName + " escuchando en puerto " + port);
    }
}
```

Crea el `Dockerfile`:

```dockerfile
FROM eclipse-temurin:17-jdk-alpine AS build
WORKDIR /app
COPY App.java .
RUN javac App.java

FROM eclipse-temurin:17-jre-alpine
WORKDIR /app
COPY --from=build /app/App.class .
EXPOSE 8080
CMD ["java", "App"]
```

```powershell
docker build -t java-config-app:1.0 .
```

## 2️⃣ Crear un ConfigMap desde un manifiesto YAML

Crea el archivo `configmap.yaml`:

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: app-config
data:
  APP_NAME: "JavaConfigDemo"
  APP_ENV: "staging"
  APP_PORT: "8080"
```

```powershell
kubectl apply -f configmap.yaml
```

Inspecciona el ConfigMap:

```powershell
kubectl get configmap app-config
kubectl describe configmap app-config
kubectl get configmap app-config -o yaml
```

## 3️⃣ Crear un ConfigMap desde la línea de comandos

```powershell
kubectl create configmap app-config-cli --from-literal=APP_NAME=DesdeLinea --from-literal=APP_ENV=test
```

Crear un ConfigMap desde un archivo de propiedades:

```powershell
@"
app.titulo=Mi Aplicacion Java
app.version=2.5.1
app.modo=produccion
app.max_conexiones=100
"@ | Set-Content app.properties

kubectl create configmap app-config-file --from-file=app.properties
kubectl describe configmap app-config-file
```

## 4️⃣ Inyectar ConfigMap como variables de entorno

Crea el archivo `pod-env.yaml`:

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: java-app-env
  labels:
    app: java-config
spec:
  containers:
    - name: java-app
      image: java-config-app:1.0
      imagePullPolicy: Never
      ports:
        - containerPort: 8080
      envFrom:
        - configMapRef:
            name: app-config
```

```powershell
kubectl apply -f pod-env.yaml
kubectl logs java-app-env
```

Deberías ver: `[staging] JavaConfigDemo escuchando en puerto 8080`

Prueba la app:

```powershell
kubectl port-forward java-app-env 8080:8080
```

Abre `http://localhost:8080` — verás la configuración inyectada desde el ConfigMap. Detén con `Ctrl+C`.

## 5️⃣ Inyectar ConfigMap como archivo montado

Crea `configmap-file.yaml`:

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: app-properties-config
data:
  app.properties: |
    app.titulo=Mi Aplicacion Java K8s
    app.version=3.0.0
    app.modo=produccion
    app.max_conexiones=200
    app.timeout=30s
```

Crea `pod-volume.yaml`:

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: java-app-volume
  labels:
    app: java-config
spec:
  containers:
    - name: java-app
      image: java-config-app:1.0
      imagePullPolicy: Never
      ports:
        - containerPort: 8080
      envFrom:
        - configMapRef:
            name: app-config
      volumeMounts:
        - name: config-volume
          mountPath: /config
  volumes:
    - name: config-volume
      configMap:
        name: app-properties-config
```

```powershell
kubectl apply -f configmap-file.yaml
kubectl apply -f pod-volume.yaml
```

Verifica que el archivo se montó correctamente:

```powershell
kubectl exec java-app-volume -- cat /config/app.properties
```

Prueba el endpoint:

```powershell
kubectl port-forward java-app-volume 8080:8080
```

Abre `http://localhost:8080/file-config` — verás el contenido del archivo montado. Detén con `Ctrl+C`.

## 6️⃣ Crear Secrets

### Desde línea de comandos (recomendado)

```powershell
kubectl create secret generic db-credentials --from-literal=DB_USER=admin --from-literal=DB_PASSWORD=S3cureP4ss2024
```

### Desde un manifiesto YAML

> **Nota:** En YAML los valores deben estar codificados en Base64.

```powershell
[Convert]::ToBase64String([Text.Encoding]::UTF8.GetBytes("admin"))
[Convert]::ToBase64String([Text.Encoding]::UTF8.GetBytes("S3cureP4ss2024"))
```

Crea `secret.yaml` con los valores Base64 obtenidos:

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: db-credentials-yaml
type: Opaque
data:
  DB_USER: YWRtaW4=
  DB_PASSWORD: UzNjdXJlUDRzczIwMjQ=
```

```powershell
kubectl apply -f secret.yaml
```

Inspecciona el Secret:

```powershell
kubectl get secrets
kubectl describe secret db-credentials
```

> **Importante:** `kubectl describe` no muestra los valores. Para verlos:

```powershell
kubectl get secret db-credentials -o jsonpath="{.data.DB_PASSWORD}" | ForEach-Object { [Text.Encoding]::UTF8.GetString([Convert]::FromBase64String($_)) }
```

## 7️⃣ Inyectar Secrets en un Pod

Crea `pod-secrets.yaml`:

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: java-app-secrets
  labels:
    app: java-config
spec:
  containers:
    - name: java-app
      image: java-config-app:1.0
      imagePullPolicy: Never
      ports:
        - containerPort: 8080
      envFrom:
        - configMapRef:
            name: app-config
        - secretRef:
            name: db-credentials
      volumeMounts:
        - name: config-volume
          mountPath: /config
        - name: secret-volume
          mountPath: /secrets
          readOnly: true
  volumes:
    - name: config-volume
      configMap:
        name: app-properties-config
    - name: secret-volume
      secret:
        secretName: db-credentials
        items:
          - key: DB_USER
            path: credentials.txt
```

```powershell
kubectl apply -f configmap-file.yaml
kubectl apply -f pod-secrets.yaml
```

Prueba todos los endpoints:

```powershell
kubectl port-forward java-app-secrets 8080:8080
```

- `http://localhost:8080` → Variables de entorno (ConfigMap + Secret)
- `http://localhost:8080/file-config` → Configuración desde archivo montado
- `http://localhost:8080/file-secret` → Secret montado como archivo

Detén con `Ctrl+C`.

**Comparativa ConfigMap vs Secret:**

| Característica | ConfigMap | Secret |
|----------------|-----------|--------|
| Propósito | Configuración no sensible | Datos sensibles |
| Almacenamiento | Texto plano en etcd | Base64 en etcd |
| Tamaño máximo | 1 MB | 1 MB |
| Inyección | envFrom, volumeMount | envFrom, volumeMount |
| Seguridad | Sin restricciones | RBAC, cifrado en reposo |

## 8️⃣ Actualizar configuración en caliente

Modifica el ConfigMap:

```powershell
kubectl edit configmap app-properties-config
```

Cambia algún valor y guarda. Los archivos montados se actualizan automáticamente (en ~1 minuto):

```powershell
kubectl exec java-app-secrets -- cat /config/app.properties
```

> **Nota:** Las variables de entorno NO se actualizan en caliente. Requieren reiniciar el Pod.
