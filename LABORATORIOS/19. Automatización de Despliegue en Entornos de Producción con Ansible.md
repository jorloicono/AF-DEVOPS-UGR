# Laboratorio 19: Automatización de Despliegue en Entornos de Producción con Ansible

> **Objetivo:** Automatizar el ciclo de vida completo de una aplicación Java en servidores de producción simulados: compilación, despliegue, gestión del servicio, health checks, rollback y despliegues canary usando Ansible Roles.

> **Requisitos previos:**
> - Docker Desktop instalado
> - Terminal PowerShell

---

## 1️⃣ Preparar el entorno de producción simulado

Crea la carpeta del proyecto:

```powershell
mkdir ansible-produccion-lab
cd ansible-produccion-lab
```

Crea `Dockerfile.node`:

```dockerfile
FROM eclipse-temurin:17-jdk-alpine

RUN apk add --no-cache openssh python3 bash curl && \
    ssh-keygen -A && \
    adduser -D devops && \
    echo "devops:devops123" | chpasswd && \
    echo "devops ALL=(ALL) NOPASSWD:ALL" >> /etc/sudoers && \
    sed -i 's/#PasswordAuthentication.*/PasswordAuthentication yes/' /etc/ssh/sshd_config && \
    mkdir -p /opt/releases /opt/java-app /var/log/java-app

EXPOSE 22 8080
CMD ["/usr/sbin/sshd", "-D"]
```

Crea `Dockerfile.control`:

```dockerfile
FROM python:3.11-alpine

RUN apk add --no-cache openssh-client sshpass bash curl && \
    pip install --no-cache-dir ansible

RUN mkdir -p /root/.ssh && \
    ssh-keygen -t rsa -b 2048 -f /root/.ssh/id_rsa -N "" && \
    echo "StrictHostKeyChecking no" > /root/.ssh/config

WORKDIR /ansible
CMD ["sleep", "infinity"]
```

Crea `docker-compose.yml`:

## 2️⃣ Crear la aplicación Java con versionado

Crea `ansible/roles/java-app/files/App.java`:

```java
import com.sun.net.httpserver.HttpServer;
import java.net.InetSocketAddress;
import java.time.LocalDateTime;

public class App {
    static final String VERSION = System.getenv().getOrDefault("APP_VERSION", "0.0.0");

    public static void main(String[] args) throws Exception {
        String host = java.net.InetAddress.getLocalHost().getHostName();
        int port = Integer.parseInt(System.getenv().getOrDefault("APP_PORT", "8080"));

        HttpServer server = HttpServer.create(new InetSocketAddress(port), 0);

        server.createContext("/", exchange -> {
            String body = String.format(
                "{\"host\":\"%s\",\"version\":\"%s\",\"status\":\"running\",\"time\":\"%s\"}",
                host, VERSION, LocalDateTime.now()
            );
            byte[] r = body.getBytes();
            exchange.getResponseHeaders().set("Content-Type", "application/json");
            exchange.sendResponseHeaders(200, r.length);
            exchange.getResponseBody().write(r);
            exchange.getResponseBody().close();
        });

        server.createContext("/health", exchange -> {
            byte[] r = "{\"status\":\"UP\"}".getBytes();
            exchange.getResponseHeaders().set("Content-Type", "application/json");
            exchange.sendResponseHeaders(200, r.length);
            exchange.getResponseBody().write(r);
            exchange.getResponseBody().close();
        });

        server.start();
        System.out.printf("[%s] v%s iniciada en %s:%d%n", LocalDateTime.now(), VERSION, host, port);
    }
}
```

## 3️⃣ Crear el Role de despliegue

### Variables por defecto del role

Crea `ansible/roles/java-app/defaults/main.yml`:

```yaml
---
app_name: java-app
app_version: "1.0.0"
app_port: 8080
app_user: devops
app_base_dir: /opt/java-app
releases_dir: /opt/releases
log_dir: /var/log/java-app
max_releases: 3
health_check_retries: 5
health_check_delay: 3
```

### Template del script de arranque

Crea `ansible/roles/java-app/templates/start.sh.j2`:

```bash
#!/bin/bash
export APP_VERSION={{ app_version }}
export APP_PORT={{ app_port }}

APP_DIR={{ app_base_dir }}/current
LOG_FILE={{ log_dir }}/app.log
PID_FILE={{ app_base_dir }}/app.pid

echo "[$(date)] Iniciando {{ app_name }} v{{ app_version }}..." >> $LOG_FILE
cd $APP_DIR
nohup java App >> $LOG_FILE 2>&1 &
echo $! > $PID_FILE
echo "[$(date)] PID: $(cat $PID_FILE)" >> $LOG_FILE
```

### Template del script de parada

Crea `ansible/roles/java-app/templates/stop.sh.j2`:

```bash
#!/bin/bash
PID_FILE={{ app_base_dir }}/app.pid
LOG_FILE={{ log_dir }}/app.log

if [ -f "$PID_FILE" ]; then
    PID=$(cat $PID_FILE)
    if kill -0 $PID 2>/dev/null; then
        echo "[$(date)] Deteniendo PID $PID..." >> $LOG_FILE
        kill $PID
        sleep 2
        if kill -0 $PID 2>/dev/null; then
            kill -9 $PID
        fi
    fi
    rm -f $PID_FILE
fi
```

### Tareas del role

Crea `ansible/roles/java-app/tasks/main.yml`:

```yaml
---
- name: Crear directorios necesarios
  file:
    path: "{{ item }}"
    state: directory
    owner: "{{ app_user }}"
    mode: '0755'
  loop:
    - "{{ app_base_dir }}"
    - "{{ releases_dir }}"
    - "{{ log_dir }}"

- name: Crear directorio para esta release
  file:
    path: "{{ releases_dir }}/{{ app_version }}"
    state: directory
    owner: "{{ app_user }}"

- name: Copiar código fuente
  copy:
    src: App.java
    dest: "{{ releases_dir }}/{{ app_version }}/App.java"
    owner: "{{ app_user }}"

- name: Compilar la aplicación
  command: "javac App.java"
  args:
    chdir: "{{ releases_dir }}/{{ app_version }}"
  become_user: "{{ app_user }}"

- name: Detener versión actual
  shell: "{{ app_base_dir }}/stop.sh"
  args:
    executable: /bin/bash
  ignore_errors: true

- name: Actualizar enlace simbólico 'current'
  file:
    src: "{{ releases_dir }}/{{ app_version }}"
    dest: "{{ app_base_dir }}/current"
    state: link
    owner: "{{ app_user }}"
    force: true

- name: Desplegar script de arranque
  template:
    src: start.sh.j2
    dest: "{{ app_base_dir }}/start.sh"
    mode: '0755'
    owner: "{{ app_user }}"

- name: Desplegar script de parada
  template:
    src: stop.sh.j2
    dest: "{{ app_base_dir }}/stop.sh"
    mode: '0755'
    owner: "{{ app_user }}"

- name: Iniciar la aplicación
  shell: "{{ app_base_dir }}/start.sh"
  args:
    executable: /bin/bash
  become_user: "{{ app_user }}"

- name: Esperar a que la aplicación esté lista
  uri:
    url: "http://localhost:{{ app_port }}/health"
    method: GET
    status_code: 200
  register: health_result
  retries: "{{ health_check_retries }}"
  delay: "{{ health_check_delay }}"
  until: health_result.status == 200

- name: Verificar versión desplegada
  uri:
    url: "http://localhost:{{ app_port }}/"
    method: GET
    return_content: true
  register: app_response

- name: Mostrar respuesta de la app
  debug:
    msg: "{{ inventory_hostname }}: {{ app_response.content }}"

- name: Limpiar releases antiguas
  include_tasks: cleanup.yml
```

Crea `ansible/roles/java-app/tasks/cleanup.yml`:

```yaml
---
- name: Listar releases
  find:
    paths: "{{ releases_dir }}"
    file_type: directory
  register: releases

- name: Ordenar releases por fecha
  set_fact:
    releases_sorted: "{{ releases.files | sort(attribute='mtime') | map(attribute='path') | list }}"

- name: Eliminar releases antiguas (mantener últimas {{ max_releases }})
  file:
    path: "{{ item }}"
    state: absent
  loop: "{{ releases_sorted[:-max_releases] }}"
  when: releases_sorted | length > max_releases
```

### Handlers del role

Crea `ansible/roles/java-app/handlers/main.yml`:


## 5️⃣ Ejecutar el despliegue v1.0.0

Crea `ansible/deploy.yml`:

```yaml
---
- name: Desplegar aplicación Java en producción
  hosts: produccion
  become: true
  serial: 1
  roles:
    - java-app
```

> **`serial: 1`** despliega en un servidor a la vez (rolling deployment), no todos a la vez.

```powershell
docker exec ansible-control ansible-playbook -i /ansible/inventory.ini /ansible/deploy.yml
```

Verifica en cada nodo:

```powershell
docker exec prod1 curl -s http://localhost:8080
docker exec prod2 curl -s http://localhost:8080
docker exec prod3 curl -s http://localhost:8080
```

## 7️⃣ Despliegue Canary

Se despliega primero en un solo nodo para validar antes de ir al resto.

Crea `ansible/deploy-canary.yml`:

```yaml
---
- name: "Fase 1: Despliegue Canary (1 nodo)"
  hosts: canary
  become: true
  vars:
    app_version: "3.0.0"
  roles:
    - java-app
  tasks:
    - name: Verificar salud del canary
      uri:
        url: "http://localhost:{{ app_port }}/health"
        method: GET
        status_code: 200
      register: canary_health

    - name: "Canary OK - listo para continuar"
      debug:
        msg: "Canary en {{ inventory_hostname }} saludable. Procediendo con el resto."

- name: "Fase 2: Despliegue al resto de producción"
  hosts: produccion:!canary
  become: true
  serial: 1
  vars:
    app_version: "3.0.0"
  roles:
    - java-app
```

```powershell
docker exec ansible-control ansible-playbook -i /ansible/inventory.ini /ansible/deploy-canary.yml
```

## 8️⃣ Rollback automático

Crea `ansible/rollback.yml`:

```yaml
---
- name: Rollback a versión anterior
  hosts: produccion
  become: true
  serial: 1
  tasks:
    - name: Listar releases disponibles
      find:
        paths: /opt/releases
        file_type: directory
      register: releases

    - name: Ordenar por fecha (más reciente primero)
      set_fact:
        releases_sorted: "{{ releases.files | sort(attribute='mtime', reverse=true) | map(attribute='path') | list }}"

    - name: Mostrar releases disponibles
      debug:
        msg: "Releases: {{ releases_sorted }}"

    - name: Verificar que hay al menos 2 releases
      fail:
        msg: "No hay versión anterior para hacer rollback"
      when: releases_sorted | length < 2

    - name: Detener aplicación actual
      shell: /opt/java-app/stop.sh
      args:
        executable: /bin/bash
      ignore_errors: true

    - name: Cambiar enlace simbólico a versión anterior
      file:
        src: "{{ releases_sorted[1] }}"
        dest: /opt/java-app/current
        state: link
        force: true

    - name: Obtener versión del rollback
      shell: "basename {{ releases_sorted[1] }}"
      register: rollback_version
      changed_when: false

    - name: Actualizar script de arranque con versión correcta
      lineinfile:
        path: /opt/java-app/start.sh
        regexp: '^export APP_VERSION='
        line: "export APP_VERSION={{ rollback_version.stdout }}"

    - name: Iniciar versión anterior
      shell: /opt/java-app/start.sh
      args:
        executable: /bin/bash
      become_user: devops

    - name: Verificar salud
      uri:
        url: "http://localhost:8080/health"
        method: GET
        status_code: 200
      retries: 5
      delay: 3
      register: health

    - name: Rollback completado
      debug:
        msg: "Rollback a {{ rollback_version.stdout }} exitoso en {{ inventory_hostname }}"
```

Ejecuta el rollback:

```powershell
docker exec ansible-control ansible-playbook -i /ansible/inventory.ini /ansible/rollback.yml
```

Verifica que se volvió a la versión anterior:

```powershell
docker exec prod1 curl -s http://localhost:8080
```

## ✅ Resumen

En este laboratorio has aprendido a:

- Crear un **Ansible Role** completo con tasks, templates, handlers, files y defaults
- Implementar despliegues basados en **releases con enlaces simbólicos**
- Usar **`serial: 1`** para rolling deployments (un servidor a la vez)
- Realizar **health checks** automáticos después de cada despliegue
- Ejecutar **despliegues canary** (probar en un nodo antes del resto)
- Implementar **rollback automático** a la versión anterior
- Limpiar releases antiguas para gestionar el espacio en disco
